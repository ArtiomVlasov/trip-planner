# Task 1.1: User Registration

## Информация
- **Status**: DONE
- **Priority**: P1
- **User Story**: US_10.1
- **Use Case**: UC-1

## Описание
Реализовать регистрацию нового пользователя по email и паролю с валидацией входных данных и созданием учетной записи в базе.

## Требования
- Проверка формата email и минимальных требований к паролю.
- Проверка, что email не занят.
- Безопасное хранение пароля (хэш).
- Возврат понятного ответа об успехе/ошибке.

## Технические детали
- Основные файлы: `components/backend/main.py`, `components/backend/services/user_resgister.py`, `components/backend/schemas.py`, `components/backend/models.py`.
- Контракт: `POST /register`.
- При занятом email возвращать корректную бизнес-ошибку.

## Критерии приемки
- [x] Регистрация работает на валидных данных.
- [x] Дубликат email обрабатывается как ошибка.
- [x] Пароль не хранится в открытом виде.
- [x] Endpoint задокументирован и используется фронтендом.
