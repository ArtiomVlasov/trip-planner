# Task 1.2: User Login

## Информация
- **Status**: DONE
- **Priority**: P1
- **User Story**: US_10.2
- **Use Case**: UC-2

## Описание
Реализовать вход пользователя по учетным данным с выдачей токена доступа для защищенных API.

## Требования
- Проверка логина и пароля.
- Выдача токена при успешной проверке.
- Обработка неверных данных без утечки чувствительной информации.
- Совместимость с защищенными endpoint-ами.

## Технические детали
- Основные файлы: `components/backend/main.py`, `components/backend/services/user_login.py`, `components/backend/services/auth_utils.py`.
- Контракт: `POST /login`.
- Токен используется в `Authorization: Bearer <token>`.

## Критерии приемки
- [x] Успешный вход возвращает токен.
- [x] Неверные данные возвращают 401/бизнес-ошибку.
- [x] Защищенные маршруты принимают выданный токен.
- [x] Проверка покрыта базовыми тестами/ручным сценарием.
